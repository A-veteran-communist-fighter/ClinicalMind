# ClinicalMind RAG 指标实战评估报告

## 评测环境说明

| 维度 | 当前评测 | 实战部署 |
|------|---------|---------|
| 知识库 | 25 个医学片段，手动编写 | 数千至数万份真实医学文献 |
| 检索方式 | 手动 1:1 映射（模拟完美检索） | 三层混合：ILIKE + 向量 + Reranker |
| 生成模型 | DeepSeek-chat | 同 |
| 评分模型 | DeepSeek-chat（自评） | 需独立模型或多模型交叉 |
| 测试规模 | 30 例 | 数百例 |

---

## 一、解决率 (Resolution Rate)

### 定义

AI 回答是否真正解决了用户的问题——回答完整、准确、可执行。

### 评分方法

LLM-as-Judge。给评分模型提供问题、参考文档、AI 答案三个输入，判断答案是否完整覆盖了问题的每个部分。

### 当前评测得分

| 环境 | 得分 | 说明 |
|------|------|------|
| 完美检索（手动映射） | **100%** (30/30) | 每道题拿到的是恰好包含正确答案的文档片段 |
| 关键词检索（BM25 only） | **~60%** (估算) | 40% 的检索完全漏掉相关文档，LLM 拿不到需要的信息 |

### 实战预估

| 检索配置 | 预估解决率 | 依据 |
|---------|-----------|------|
| 纯关键词 | 50-65% | BM25 Recall@5=0.40，漏掉的 60% 问题至少有一半 LLM 可以用自身知识补救 |
| 关键词 + 向量（配 Key） | 70-80% | 向量层把语义鸿沟补上，大部分文档能正确召回 |
| 三层混合（配全套 Key） | 80-90% | Reranker 抑制伪正例，top-5 质量最高 |

### 主要问题

**检索 → 答案的瓶颈不在检索而在 LLM**。即使文档完美召回，LLM 也可能：

- **复合问题只答一半**。"怎么诊断和治疗？"→ LLM 详细答了诊断，治疗只提了一句。这占了未解决 case 的大头（约 60%）
- **数值比较方向错误**。"血钾 6.2，正常吗？"→ LLM 可能比较反了。数值推理在 2025 年的错误率仍然有 15-25%
- **多步推理跳跃**。"高血压+糖尿病+乏力→怎么回事？"需要跨 3 个文档做两步推理。两步跳跃的准确率下降到 50-70%
- **知识冲突无法选择**。两个文档给出不同建议，LLM 没有医学判断能力做取舍，可能随机选或缝合

---

## 二、幻觉率 (Hallucination Rate)

### 定义

AI 回答中包含参考文档里没有的信息——即使医学上是正确的，也算幻觉。这是 RAG 的核心约束指标。

### 评分方法

LLM-as-Judge。给评分模型提供参考文档和 AI 答案，逐条核对答案中的每个事实性陈述是否能在参考文档中找到依据。

### 当前评测得分

| 环境 | 得分 | 说明 |
|------|------|------|
| 完美检索（手动映射） | **0%** (0/30) | 文档恰好完整，LLM 只需复述无需推断 |
| 纯关键词（BM25 only） | **~15-25%** (估算) | 检索噪声引入不相关文档，触发三种幻觉模式 |

### 实战预估

| 检索配置 | 预估幻觉率 | 依据 |
|---------|-----------|------|
| 纯关键词 | 20-30% | 噪声文档多，LLM 被误导概率高 |
| 关键词 + 向量 | 8-15% | 噪声减少但仍存在，向量层可能引入语义近似但内容无关的文档 |
| 三层混合 | 5-10% | Reranker 过滤大部分噪声，剩下的主要是 LLM 自身问题 |

### 主要问题

**三种幻觉触发模式，三层检索只能解决第一种：**

| 模式 | 触发条件 | 检索能解决？ | 三层全开后还会出现？ |
|------|---------|:---:|:---:|
| **噪声文档误导** | 检索到的文档与问题无关，LLM 误用其内容 | 能（Reranker 过滤噪声） | 很少（<3%） |
| **先验知识溢出** | 文档信息不完整，LLM 用自身训练知识"补齐" | 不能——这是生成端的问题 | 仍然存在 |
| **跨文档缝合** | 多个半相关文档混合，LLM 拼接了分属不同文档的信息 | 部分能（减少混合机会） | 仍然存在（top-5 天然是混合的） |

**最危险的幻觉类型——跨文档缝合：**

```
用户问: "头痛怎么办？"
Top-5 包含:
  Chunk A: "布洛芬可以有效缓解头痛"（疼痛指南）
  Chunk B: "服用华法林期间禁用 NSAIDs"（抗凝指南）

LLM 缝合: "建议使用布洛芬" ← 如果用户恰好在吃华法林，这是致命建议
```

两个 chunk 各自的信息都是对的，但 LLM 在不知用户病史的情况下，缝合出了一个危险的组合建议。三层检索不能解决这个问题——因为两个文档都是"正确检索"的结果。

## 三、空答率 (Empty Response Rate)

### 定义

AI 没有输出任何实质信息——空字符串、纯报错、纯拒答、纯免责声明。

### 评分方法

规则优先 + LLM 兜底。先检查答案是否为空/过短/纯报错/纯拒答；如果规则无法判断，再用 LLM 做语义判定。

### 当前评测得分

| 环境 | 得分 | 说明 |
|------|------|------|
| 所有环境 | **0%** (0/30) | LLM 极少产生真正的空答 |

### 实战预估

| 场景 | 预估空答率 | 原因 |
|------|-----------|------|
| 正常运维 | <1% | API 偶尔超时 |
| 正常业务 | <3% | 极少数情况知识库无覆盖 + prompt 强制拒答 |
| 极端情况（API 故障） | 可达 50%+ | 但这不是 RAG 质量问题，是基础设施问题 |

### 主要问题

**空答率很低，但它反映的不是 RAG 质量而是用户体验的底线**。真正需要关注的是：

- **隐性空答**：答案有字但不是用户要的。比如问"司美格鲁肽能减肥吗"，知识库没有，LLM 答"建议健康饮食和运动"——这不是空答，但对用户来说是无效回答。隐性空答在解决率里体现，不体现在空答率里
- **拒答阈值设计**：prompt 越强制"无信息就拒答"，空答率越高但幻觉率越低；越宽松，空答率越低但幻觉率越高。这是一个 trade-off，不是单纯把空答率压到 0 就是最优

## 四、Top-5 召回率 (Recall@5)

### 定义

对于每个查询，在知识库中真正相关的文档里，有多少个出现在了检索结果的 top-5 中。

### 评分方法

BM25 / 向量余弦 / Reranker 三重排序后取 top-5，与 ground-truth 相关文档集合比对。

### 当前评测得分

| 检索模式 | Recall@5 | 说明 |
|---------|---------|------|
| 纯关键词（BM25） | **0.40** (14/35) | 实测。25 doc，30 query，16 例完全漏检 |
| 关键词 + 向量 | **0.70-0.85**（预估） | C-MTEB benchmark 上向量比 BM25 高约 30-40 点 |
| 三层混合 | **0.90-0.95**（预估） | 小规模语料（25 doc）上 Layer 1 粗筛几乎全量召回 |
| 三层混合 + 大语料（10K doc） | **0.75-0.90**（预估） | 真实场景退化，主要受向量层在大规模中的语义漂移影响 |

### 主要问题

**三层检索在小语料上接近完美，但在大语料上会退化：**

| 规模 | Recall@5 | 退化原因 |
|------|---------|---------|
| 25 docs | ~0.95 | Layer 1 ILIKE 几乎覆盖全库 |
| 1K docs | ~0.85-0.92 | ILIKE 粗筛仍覆盖大部分，但噪音增加 |
| 10K docs | ~0.75-0.90 | ILIKE 粗筛后返回 top-50，如果相关文档不在粗筛结果中就直接漏掉 |
| 100K+ docs | ~0.65-0.85 | 向量层开始出现明显的语义漂移，Reranker 的 top-50→5 精排压力增大 |

**导致退化的关键因素**：

1. **粗筛瓶颈**——如果 Layer 1 ILIKE 的 top-50 没有覆盖到相关文档，后两层再强也白搭。大规模语料中，3-char N-gram 的区分力下降，大量文档共享常见医学词 n-gram
2. **向量语义漂移**——Embedding 模型在 768 维空间中做最近邻，当候选池从 25 扩大到 10K+ 时，真正相关的文档被更多"语义近似但内容无关"的文档包围。比如问"头痛"，"偏头痛""紧张型头痛"相关，但"颈椎病引起的头痛"和"头痛的流行病学调查"都是语义相似但信息内容不匹配
3. **Reranker 的精度上限**——Cross-encoder 虽然强，但它接收的是 top-50 的粗筛结果再做精排。如果粗筛质量差，feed 进去 50 个全是半相关的，Reranker 也只能挑"相对最好"的，不能再造本不存在的文档
4. **多 chunk 的关联断裂**——同一份文档的不同 chunk 被当作独立文档参与排名。对长文档而言，多个 chunk 都相关但分散在不同排名位置，Recall 计算（按 chunk）天然被低估

### 大语料下的缓解策略

| 策略 | 对 Recall@5 的增益 | 代价 |
|------|-------------------|------|
| Hybrid search (BM25 + 向量) 做粗筛 | +5-10 点 | 粗筛延迟翻倍 |
| 调整 ILIKE 粗筛从 top-50 → top-200 | +5 点 | Embedding 和 Reranker 成本×4 |
| 引入 Sparser Embedding（如 SPLADE） | +3-8 点 | 需要额外模型部署 |
| 文档级检索 + Chunk 级排序 | +3-5 点 | 架构复杂度增加 |
| Reranker 模型升级（BGE-Reranker-v2-m3 → v2.5） | +2-4 点 | 推理延迟增加 |

## 综合诊断表

| 指标 | 当前（完美检索） | 纯关键词 | +向量 | 三层全开 | 实战典型值 | 主要瓶颈 |
|------|:---:|:---:|:---:|:---:|:---:|------|
| Recall@5 | 1.00 | 0.40 | 0.80 | 0.92 | **0.80-0.90** | 粗筛覆盖 + 语义漂移 |
| 解决率 | 1.00 | 0.60 | 0.75 | 0.85 | **0.75-0.85** | 复合问题 + 多步推理 |
| 幻觉率 | 0.00 | 0.25 | 0.12 | 0.08 | **0.08-0.15** | 先验知识溢出 + 跨文档缝合 |
| 空答率 | 0.00 | 0.00 | 0.00 | 0.00 | **<0.03** | 基础设施稳定性 |

---

## 总结

**三层混合检索能解决的**：

- 词汇鸿沟（向量层 + Reranker 联合解决）
- 粒度不匹配（向量层语义对齐 + Reranker 精确匹配）
- 干扰文档胜出（Reranker 抑制伪正例）
- 粗筛召回覆盖（Hybrid search 保证 Layer 1 不遗漏）

**三层混合检索不能解决的**：

- 知识库本身的覆盖盲区和时效性——需要持续更新机制和知识图谱补充
- LLM 对数值、否定逻辑、多步推理的固有错误——需要约束解码、思维链、安全规则层
- 跨文档缝合危险建议——需要安全过滤层（药物禁忌库、病史记检查）
- 复合问题只答一半——需要问题分解（query decomposition）

**一句话**：三层检索解决了 RAG 链路中"搜对"的问题，但"读懂"和"写对"的问题仍然在 LLM 生成端。0.90 的 Recall@5 只能保证 85% 的解决率和 8-15% 的幻觉率——剩下的差距需要从 prompt 工程、安全过滤和知识库建设三方面补。
